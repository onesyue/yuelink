```bash

git add lib/modules/emby/emby_player_page.dart
git commit -m "fix(emby): resume from saved position no longer races with autoplay

_player.open(media) defaulted to play: true, so the player started
from 0 immediately while _checkResume() was still waiting on the
Emby /UserData REST round-trip + the user's dialog click. By the

raced with the autoplay/buffer pipeline and frequently got reset
to 0 — the dialog felt broken.

the paused player — no race), then call _player.play(). Single
deterministic start point."
```

## 4. Tag / Push 流程（CLAUDE.md `Git & CI` 段）

**重要 invariants**：
* `master` 是发布分支，`dev` 是开发分支
* CI 只在 tag + workflow_dispatch 触发
* `pre` tag 是预发布（覆盖前一个）
* **从不**在 push commit 之前 tag

```bash
# 1. 17 个 commit 合到 dev
git checkout dev
git pull origin dev
# (按上面 C1-C17 顺序 commit；本仓库 git status 显示在 master 上,
#  实际操作时按你的工作流程切到 dev 分支再 commit)
git push origin dev

# 2. （可选）移动 pre tag 出测试构建
git tag -d pre 2>/dev/null
git push origin :refs/tags/pre 2>/dev/null
git tag pre
git push origin pre
# → CI 触发 prerelease build，等 GitHub Actions 全绿

# 3. 真机回归（这一步是合并门槛 — 见 § 5）

# 4. 合 dev → master
git checkout master
git merge --no-ff dev -m "Merge dev into master for v1.1.21 release"
git push origin master

# 5. 打正式 tag
git tag -a v1.1.21 -m "v1.1.21 — iOS LAN 直连 + 国内银行 fake-IP 修复 + Emby 继续播放修复"
git push origin v1.1.21
# → CI 触发正式 release build
```

> ⚠️ **CLAUDE.md 锁的红线**：
> * 不要在我（Claude）这边自动 `git push origin <tag>` —— memory `feedback_no_auto_push` 明确"正式版/tag 必须用户明确同意才能推"
> * 不要 `git push --force-with-lease origin master`，master 是发布分支
> * 不要在 GitHub Release 里手动改 release notes —— `docs/releases/v1.1.21.md` 是单一真源

## 6. 回滚预案

如果 v1.1.21 发布后 24 小时内出现严重 regression：

* **iOS LAN 排除导致 X 服务不通**：iOS 用户卸载 v1.1.21 → 装 v1.1.20。代码层面下个版本 `c.P3-2` 加 `excludedRoutes` 用户开关（默认关）。
* **Emby 继续播放反向 regression**（点继续不播）：在 dev 分支 revert C15 + 出 v1.1.21.1 hotfix。
* **Windows TUN 热切换 regression**：revert C5 + C6 + C7（保 catalog / a 不动）+ 出 v1.1.21.1。
* **DNS 泄漏复发**：revert C2 + C3，保留 C1 catalog 抽取（无行为变化）。

每个 revert 范围已经按 commit 边界设计好，cherry-pick / revert 单 commit 即可。
