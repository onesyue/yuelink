## Fixes

- **看 YouTube / 视频流更稳，首页不再"暗中抢带宽"** — 主屏底部的悦视频推荐栏改为**按需加载**：只有你主动进过一次悦视频，下次回首页才会拉真实海报；之前不论用不用，每次回首页都会向你的悦视频服务器**并发** 2–5 个查询请求，跟前台流量（YouTube/Netflix/会议/直播）抢同一个出口节点的带宽和连接池——YouTube 这种对短期吞吐敏感的应用会被这股暗流挤到降码率甚至卡顿。同步策略对其它两条暗流也收紧：
  - **悦视频会员状态在 24h 缓存有效期内不再后台二次校验**——之前每次进首页都会无声向账户系统重新验一次，缓存命中也照发；
  - **匿名用量上报失败后进入 90s 退避**——服务器抖动时不再每几秒撞一次代理重试，正常 60s 周期上报不受影响。
- **真机实测对比（同一 Korea 节点 / 同一 YouTube 视频）**：修复前主屏停留时进程 CPU 占用 21.4%、`TIME_WAIT` 残留 29 条；修复后 CPU 降至 3.4%（↓6×）、`TIME_WAIT` 仅 7 条（↓4×）；YouTube 视频流不再因后台请求挤压而 buffer。

## Upgrade Notes

- 已登录用户无需重登。登录态、订阅、悦视频凭证、播放记录、本地设置全部保留。
- **从没用过悦视频的用户**：首页那行海报第一次会显示**灰色占位图**（视觉上跟 v1.1.17 网络异常时的占位完全一致），点一下进入悦视频后下次回首页就恢复真实海报，且永久生效。
- **正在用悦视频的用户**：切到这一版后 24h 缓存窗口内不会主动重拉会员信息，到期后恢复原有逻辑。会员开通 / 到期 / 切换等状态变化在缓存窗口内可能有最长 24h 延迟感知；如需立即刷新可手动退出登录再登录。
- 没改任何 mihomo 内核 / TUN / DNS / 路由配置；订阅 YAML、节点选择、规则集与 v1.1.17 完全一致。
