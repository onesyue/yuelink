## Fixes

- 看 YouTube / 视频流更稳，首页不再"暗中抢带宽"。主屏底部的悦视频推荐栏改为按需加载：只有主动进过一次悦视频，下次回首页才会拉真实海报。之前不论用不用，每次回首页都会向悦视频服务器并发 2–5 个查询请求，跟前台 YouTube / Netflix / 会议 / 直播抢同一个出口节点的带宽和连接池。
- 悦视频会员状态在 24 小时缓存窗口内不再后台二次校验。
- 匿名用量上报失败后进入 90s 退避，服务器抖动时不再每几秒撞一次代理重试，正常 60s 周期上报不受影响。

真机实测对比（同一韩国节点 / 同一 YouTube 视频）：修复前主屏停留时进程 CPU 占用 21.4%、TIME_WAIT 残留 29 条；修复后 CPU 降至 3.4%、TIME_WAIT 仅 7 条；YouTube 视频流不再因后台请求挤压而 buffer。

## Upgrade Notes

- 已登录用户无需重登。登录态、订阅、悦视频凭证、播放记录、本地设置全部保留。
- 从没用过悦视频的用户：首页那行海报第一次显示灰色占位图（视觉上跟 v1.1.17 网络异常时的占位一致），点一下进入悦视频后下次回首页恢复真实海报，永久生效。
- 正在用悦视频的用户：切到这一版后 24 小时缓存窗口内不会主动重拉会员信息，到期后恢复原有逻辑。会员开通 / 到期 / 切换在缓存窗口内可能有最长 24 小时延迟感知，如需立即刷新可手动退出登录再登录。
- 没有改任何 mihomo 内核 / TUN / DNS / 路由配置；订阅 YAML、节点选择、规则集与 v1.1.17 完全一致。
