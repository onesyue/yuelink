## What's New

- **新用户默认进入访客模式** — 不登录也能先看 Dashboard / 节点页 / 设置；要付费节点 / 订阅 / Emby 时 Dashboard 顶部弹登录 CTA，不再上来就拦在登录页。
- **iOS 首次启动 VPN 权限解释** — 系统弹「YueLink 想添加 VPN 配置」之前，先用一个 in-app 解释卡说清楚为什么需要、点同意后会发生什么，避免误拒。
- **macOS Gatekeeper 自动修复** — 从 DMG 拖到应用程序后，下次启动若仍被 quarantine flag 拦下，app 会自检并静默修复，不再要求用户手动跑 `xattr -d`；Settings 顶部也提供一键修复入口兜底。
- **Linux AppImage 内置 libmpv** — Ubuntu 20.04 / Debian 11 / 旧版 Fedora 不用再手动 `apt install libmpv2`，AppImage 解压即跑（Issue #3）。
- **首次连接成功有明确提示** — 桌面系统代理模式连上后弹一次「系统代理已启用 — 所有流量将通过 YueLink 路由」；任何模式连上都会做一次连通性自检并 toast 反馈，避免「明明连上了但没反馈」的盲区。
- **iOS 连接期间显示进度遮罩** — 启动隧道时遮罩 + 进度文案，避免「点了但没反应」的盲区。
- **Android 电池豁免申请前先弹 in-app 说明** — 进入 Settings → 连接修复申请白名单时，先解释为什么需要、紧接着的系统对话框会怎么写、能不能省电，再唤起 OS 弹窗。小米 / 华为 / OPPO 用户不再面对孤零零的系统对话框困惑。
- **订阅同步失败弹独立对话框** — 替代以前一闪而过的红色 toast，列出错误原因 + 「重试 / 切换订阅 / 查看日志」三个选项。
- **Emby 标签页冷启动即时显示** — 上次的服务器地址 / 用户 ID / Token 在登录后被本地缓存（24h TTL），再开 app 时 Emby tab 不再先白屏一两秒等远程 sync。
- **Emby 订阅用户首次启动有引导卡** — 订阅自带 Emby 账号的用户在 Dashboard 看到激活按钮，不用翻设置。
- **Emby 库类型支持 STRM 服务器** — 视频 / 影集库能正确识别 `Movie,Video` / `Series,Video`，搬运服 Emby 节目数不再为 0。

## Performance

- **心跳按网络类型分档** — Wi-Fi 15 秒 / 蜂窝 30 秒，前后台再分一档（60 秒 / 120 秒）。手机端在地铁、移动热点、长时间挂后台时电量明显改善。
- **代理组结构性比较 + 列表短路** — `/proxies` 轮询返回相同内容时整条 UI rebuild 链短路，节点页在大订阅下滚动更顺。
- **后台时日志流自动暂停** — 切到后台时 logs WebSocket 暂停，前台恢复，节省后台 socket / 电量；状态保留，回前台不掉之前的内容。
- **订阅下载启用 gzip** — 显式 `Accept-Encoding: gzip, deflate`，原生支持的服务端会真的压缩，订阅同步流量减少。
- **mihomo 升级到 v1.19.24** — 跟上游 stable 走，包含 QUIC race / dialer race / TUN 启动文件检查 / TLS CVE-2025-68121 等修复。

## Bug Fixes

- **安卓 / Windows 启动后节点列表不再无限「处理中」转圈** — 大订阅（200+ 节点）的 `/proxies` 响应过早被下沉到背景 isolate，闭包捕获 String body 在 release 包上间歇挂死，被静默吞掉，节点图永远空。阈值回调到 256 KB 让典型响应留在主线程；同时 `ProxyGroupsNotifier` 在状态已是 running 时主动补一次刷新，覆盖热启动 / resume 漏接监听的边界。
- **rule-providers 不再被订阅的兜底分流死锁** — 改用 mihomo 原生方案，给每个未声明 fetch 路由的 provider 自动注入 `proxy: DIRECT`。覆盖任意 CDN（jsdelivr / fastly / githubusercontent / gcore / 自建镜像），不再污染用户的 rules 列表；用户自己写了 `proxy: <group>` 的会被尊重。
- **启动 5 秒超时不再误报 E007** — `waitProxies` 预算从 5 秒放到 15 秒，且超时改为软着陆（启动报告里显示 `slow`）。订阅含 `proxy-providers` / `rule-providers` 第一次拉网络资源时不会再被红 banner 吓到。
- **macOS 端口冲突时启动不再 verify 失败** — 9090 被占自动改 9091 后，恢复路径不会再把内存里的 9091 用旧持久化值 9090 覆盖回去（之前 E008_CORE_DIED_AFTER_START 的根因）。
- **连接成功后 UI 不再显示「未连接」** — 三个并发路径合修：手动停止标志的写盘改即时刷盘、resume handler 在 `start()` 进行中整体 defer、Dashboard hero card 在节点列表加载中显示「处理中」而不是骗用户的「未连接」。
- **start() 失败的清理路径不再卡死后续操作** — 启动失败时如果写诊断报告自身又抛异常，老逻辑会跳过清理步骤，导致下一次连接 / resume 永久卡住直到重启 app。改成 finally 兜底，不论中间怎么抛都释放守门标志。
- **节点列表在 graph 还没就绪时自动重试** — 进 Dashboard 后即使 mihomo 还没把代理图组装好，每 3 秒静默 retry 一次直到拿到，无需手动下拉刷新。
- **Emby 首页推荐拉取「Client is already closed」** — Provider 被父级失效瞬间 close 了正在飞的请求；改为在请求自身 finally 中关闭。
- **iOS / Android 原生 MethodChannel 调用加超时** — VPN 启停、托盘、Quick Settings Tile 不再有概率永久挂起 await。
- **诊断日志包含已轮转的 core.log.1 / .2** — 导出时自动带上历史日志，复现的事件不会因日志旋转丢失上下文。

## Upgrade Notes

- 已登录用户无需重登，订阅与设置自动保留。
- macOS：DMG 直接覆盖即可；之前因为没签名要手动跑命令解隔离的，这版会自动处理。
- Linux：AppImage 已自带 libmpv，不再需要 `apt install libmpv2`；旧版可以直接覆盖 1.0.23 二进制。
- Windows / Android：直接安装新版，「处理中」无限转圈的回归已修复。
- iOS：TrollStore / AltStore / SideStore 重装新 IPA 即可；首次启动会看到 VPN 权限解释卡，点继续即可。
- 用过 1.0.23-pre 的：差异主要是「处理中转圈」「rule-providers 死锁」「启动失败导致 app 卡死」三个回归已修，正式版可以直接覆盖。
