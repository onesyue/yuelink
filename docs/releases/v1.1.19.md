## Fixes

- TUN 模式下 ChatGPT、Claude 卡在 Cloudflare"Just a moment"验证页或一直转圈，切回系统代理就正常。原因是浏览器开了 Secure DNS（Chrome 默认开）或 ECH 后，Cloudflare 给所有走它的网站统一用一个外层 SNI（cloudflare-ech.com），sniffer 看到这个伪装名后把整条 TLS 连接路由给兜底分流，而 DoH / ECH 探测又落到另一条出口；Cloudflare 收到两边来自不同地区的 client IP 之后判定为可疑会话，弹验证墙。系统代理模式下浏览器自检测到代理会自动关 Secure DNS，所以问题被掩盖、只在 TUN 下显形。
  - 客户端：sniffer 跳过 cloudflare-ech.com，遇到 ECH 外层 SNI 改用 fake-ip 反查得到的真实业务域名（chatgpt.com / claude.ai 各自）路由；同时把 cloudflare-dns.com / chrome.cloudflare-dns.com / mozilla.cloudflare-dns.com / dns.google / cloudflare-ech.com 自动前置到当前订阅的"主代理组"（智能识别：AI / ChatGPT / OpenAI / Claude / Gemini 类组优先；否则 GLOBAL / PROXY / AUTO / 节点选择 / 🚀 / 手动切换 / 自动选择 / 全部节点 兜底；都没识别则不强行注入，留给手工覆盖）。这两步加起来让 DoH 探测、ECH 探测、主业务连接走同一个出口节点，Cloudflare 看到一致 client IP 就直接放行。
  - 订阅模板：服务端同步加这套规则，让用同一份订阅的第三方 clash / clash-verge / clash-party / Stash 用户也享受修复。订阅模板顺手剔除了 parse-pure-ip / force-dns-mapping 两个开关，第三方核心实测吞吐回升约 30%。
  - 副作用：所有走 Cloudflare 的网站（ChatGPT / Claude / GitHub / Discord / Reddit / Notion / Vercel / …）开 ECH 时一并受益，无需逐站点配规则。
- 单个节点持续失败时，相同节点 / 探测目标 / 错误类型组合的失败事件 5 分钟内最多上传 3 条，超出窗口跳过；不再因为一个挂掉的节点把上传缓冲打满，正常事件不会被挤出。

## Upgrade Notes

- 已登录用户无需重登，订阅、设置、节点选择全部保留。
- ECH 路由修复全量自动生效，不需要手动调任何开关；浏览器 Secure DNS 可以保持开启状态。
- 这版唯一会影响订阅 YAML 形态的是 cloudflare-dns.com / cloudflare-ech.com 等域名被自动注入到主代理组规则。如果在覆盖配置里手动给这些域名设过其它路由，覆盖优先级高于本版注入，原设置继续生效。
- 没有改 mihomo 内核版本、TUN 配置、DNS 配置、节点选择口径；没有触碰 v1.1.18 的悦视频懒加载策略。
