## Fixes

- **修复 ChatGPT、Claude 在 TUN 模式下打不开** — 浏览器开了 Secure DNS（Chrome 默认开）或 ECH 后，访问 ChatGPT / Claude 会卡在 Cloudflare 的"Just a moment"验证页或一直转圈，但切回系统代理就正常。原因是 Cloudflare 给所有走它的网站统一伪装一个 outer SNI（`cloudflare-ech.com`），sniffer 看到这个伪装名后把整个 TLS 连接路由给"兜底分流"，而 DoH/ECH 探测又落到另一条出口；Cloudflare 收到两边来自不同地区的 client IP 之后判定为可疑会话→出验证墙。系统代理模式下 Chrome 自检测到代理会自动关 Secure DNS，所以问题被掩盖，只在 TUN 下显形。
  - **客户端**：sniffer `skip-domain` 加 `cloudflare-ech.com`，遇到 ECH outer SNI 时改用 fake-ip 反查得到的真实业务域名（chatgpt.com / claude.ai 各自）路由；同时把 `cloudflare-dns.com` / `chrome.cloudflare-dns.com` / `mozilla.cloudflare-dns.com` / `dns.google` / `cloudflare-ech.com` 自动前置到当前订阅里的"主代理组"（智能识别：AI / ChatGPT / OpenAI / Claude / Gemini 类组优先；否则 GLOBAL / PROXY / AUTO / 节点选择 / 🚀 / 手动切换 / 自动选择 / 全部节点 兜底；都没识别则不强行注入，留给用户手工 overwrite）。这两步加起来让 DoH 探测、ECH 探测、主业务连接走同一个出口节点 → Cloudflare 看到一致 client IP → 直接放行，不再出墙。
  - **机场订阅模板**：服务端 `app.clash.yaml` / `default.clash.yaml` 同步加这套规则，让用同一份订阅的第三方 clash / clash-verge / clash-party / Stash 用户也享受修复。订阅模板顺手剔除了 `parse-pure-ip: true` / `force-dns-mapping: true`，第三方核心实测吞吐回升约 30%（这两个开关 v1.0.21 P1-4 已在 yuelink 客户端模板里去掉，这次补到服务端订阅）。
  - **副作用**：所有 cf 后的网站（ChatGPT / Claude / GitHub / Discord / Reddit / Notion / Vercel / ...）开 ECH 时一并受益，无需逐站点配规则；浏览器侧不需要关 Secure DNS。

- **后台失败探测不再撑爆事件队列** — 单个节点持续失败时，相同 `(节点, 探测目标, 错误类型)` 的失败事件在 5 分钟内最多上传 3 条；超出窗口后跳过，不再因为一个挂掉的节点把上传缓冲打满，正常事件不会被挤出。

## Internal

- **遥测维度扩展**（仅对开启「匿名遥测」的用户生效，默认关）：
  - 节点探测事件 `node_probe_result_v1` 现在带 `xb_server_id`（XBoard 节点真实 ID），服务端用它从 `nodes-inventory-path-map.json` 推 `path_class`（直连 / via_v4_relay / via_v6_relay），不再因为多个节点共用 `host=v4.yuetoto.net` 主机名而被错归一类。

- 已登录用户无需重登，订阅、设置、节点选择全部保留。
