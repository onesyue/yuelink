## New

- 启动时"发现新版本"通知改为可点击：点一下通知胶囊直接弹出更新对话框（含发版说明 + 下载按钮），不必再绕到 我的 → 检查更新。
- 更新下载从单点直连升级到多镜像链：优先走 gh-proxy.com / ghfast.top 两个 CDN，原直连作兜底；每条镜像 15s 连接超时 + 20s 首字节超时 + 15s 无进度看门狗，卡住的镜像自动切换。国内下载预期 5–20 MB/s。
- macOS 自动安装新版：下载 .dmg 后无需手动拖拽。挂载 dmg → 拷贝 YueLink.app 到 /Applications → 清理 macOS 隔离标记 → 卸载 dmg → 重启新版本。任一步失败会回退到原"打开 dmg 让用户拖"流程。
- 桌面 hosts 文件支持：默认开启 use-system-hosts，自定义 `/etc/hosts` 条目优先于 fake-IP。

## Fixes

- 桌面 TUN ↔ 系统代理切换不再"卡端口"：旧 mihomo 进程退出后的 7890（mixed-port）/ 1053（DNS）/ 控制器端口立即释放，新核心起来时不再 bind 失败、不再出现"界面看到的出口节点跟实际访问的不是同一个"。
- macOS / BSD 在 IPv6 wildcard 已被占用时仍允许 bind 127.0.0.1 表面成功导致的端口冲突假阴性：现在同时探测 127.0.0.1 与 0.0.0.0，任一失败即视为占用。
- 诊断报告新增 mihomo 子进程日志：桌面 TUN 模式下 mihomo 由特权 helper 启动的子进程日志（mihomo-service.log）一并打包，排查 TUN 故障不需要二次往返。

## Changed

- DNS 缓存升级到 ARC 算法：替代之前的 LRU。代理客户端的 DNS 工作负载里，热点域名 + 长尾一次性查询是典型组合，ARC 在这类负载下命中率比 LRU 高 20–40%。

## Upgrade Notes

- 已登录用户无需重登。登录态、订阅、Emby、播放记录、本地设置全部保留。
- macOS 用户从 v1.1.16 升上来时如果之前 TUN 切换留下了僵尸 mihomo 进程，本版第一次启动前手动确认下：托盘退出 YueLink → `pkill -9 yuelink-mihomo yuelink-service-helper` → 再启动新版。装上 v1.1.17 之后切换都不会再出现这种残留。
