# YueLink v1.1.0-pre

预发布版，2026-04-30。本轮聚焦 UI 精细化、Android 启动稳定性、悦视频代理路径修复，以及节点遥测身份上报。

## What's New

- **节点匿名指纹 v2**：订阅解析后上报 `node_inventory.nodes`，每个节点带 `fp / type / xb_server_id`，缺少数字 ID 时回退 `sid`。`node_urltest` 使用同一份 fp 缓存，测速结果可和 inventory 精确关联。
- **iOS 26 风格继续统一**：首页 / 线路 / 悦视频 / 我的移除顶部大字标题，页面留白、字体、列表、底部 tab 更贴近 Apple 通讯录 / 电话 app 的密度和层级。
- **系统字体栈**：桌面和移动端改用平台系统字体，移除 Google Fonts 依赖，降低启动和离线字体风险。
- **悦视频代理感知**：媒体库和图片请求只在核心运行时走本地代理，未连接时不再请求 `127.0.0.1` 导致 connection refused。
- **Feature flags 启动顺序**：flags 请求等匿名 `client_id` 初始化完成后再发，空 client_id 会被跳过。

## Bug Fixes

- **Android 启动卡死 / 无响应**：移除冷启动时的 eager FlutterEngine 预热，改为按需缓存，降低 Android 原生启动阶段阻塞。
- **底部 tab 遮挡页面**：底部导航和页面内容参与同一布局，避免盖住列表末尾。
- **测速失败事件补 reason**：失败测速标准化为 `delay_ms=5000` + `reason=timeout`，同时携带节点身份字段。
- **telemetry sanitizer 修复**：`List<Map<String, scalar>>` 不再被清空，`node_inventory.nodes` 能真实送到服务端。
- **悦视频错误提示**：本地代理未启动导致的 127.0.0.1 refused 会显示连接提示，不再误判为 UI 卡死。

## Guardrails

- 增加回归测试覆盖 v2 fp 合约、`sid` hash fallback、WebSocket Host fallback、inventory 不泄露 server/port/password/sni、urltest 与 inventory fp 一致、失败 reason、flags URI。
- 相关 telemetry / 节点 / mihomo API / purchase / crash dedup 测试已通过。

## Upgrade Notes

- 已登录用户无需重登，订阅与设置自动保留。
- Android：直接安装新版 APK。若之前遇到启动无响应，建议覆盖安装后冷启动一次验证。
- macOS / Windows / Linux：直接覆盖。
- iOS：AltStore / SideStore 用户重装新 IPA 即可。
