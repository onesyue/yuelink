## What's New

- **主页徽章悬停提示** — 「规则 / 全局 / 直连」「TUN / 系统代理」徽章悬停或长按时显示「点击切换」，避免反馈中"不知道徽章可以点"的盲区。
- **托盘悬停显示当前模式** — 鼠标移到任务栏图标上，提示直接显示「YueLink · 已连接 · 规则 · TUN · 节点名」，不开主界面就能看清当前路由和接入方式。

## Performance

- **Windows 下载稳定 + 吞吐对齐** — `find-process-mode` 默认值改为 strict（与 mihomo 上游一致）。下载工具频繁创建子进程时不再每个连接都触发系统调用，迅雷 / IDM / Steam 大文件下载更稳，吞吐回到与裸 mihomo 相同的水平。订阅显式设置的值仍被尊重。

## Bug Fixes

- **桌面 TUN 服务模式不再「一会断连一会又连上」** — 心跳判定改用 REST API 状态而非 in-app FFI 标志。服务模式下 mihomo 在助手子进程，FFI 永远是 false，旧逻辑被错误地当成失败并触发自动恢复循环。
- **断开后切回不再显示「已连接」但实际无网** — 应用恢复期间二次校验用户停止意图；UI 状态从 derived provider 读取，绝不会出现图标显示连接而实际无网的一帧。
- **断开重连后立即测速不再「全红」** — 启动等到代理图就绪 (`/proxies` 返回非空且有 GLOBAL 或可用策略组) 才放行，重试时主动 healthcheck providers 刷新 selector 缓存。
- **Windows 11 任务栏 Close 在连接中能彻底退出** — 连接状态下点窗口 X 或任务栏 Close 直接走退出流程，不再悄悄缩到托盘后留下 `yuelink.exe` 在任务管理器。
- **冷启动不再白屏** — 启动阶段 SecureStorage / settings 读取加 4 秒超时，任何异常都不会阻止 `runApp()` 启动；登录态 unknown 时显示居中加载指示器而不是空白。
- **v2rayN / Clash Verge 劫持系统代理后立即恢复** — resume 时若一次写失败，自动 1.5 秒退避重试 3 次，不用等 30 秒心跳兜底。
- **更精准的诊断信息** — 心跳每次失败按 socket / timeout / HTTP 状态码分类写入事件日志；导出诊断时自动包含已轮转的 `core.log.1` / `.2` 历史，复现的事件不会因日志旋转丢失上下文。
- **`crash.log` 不再无限增长** — 上限 1 MB 自动归档为 `.1`，最多保留 ~2 MB；同一异常的远端上报也走与 telemetry 相同的去重窗口，不再洪流式上报。

## Upgrade Notes

- 已登录用户无需重登，订阅与设置自动保留。
- 修复全部在客户端侧，必须升级到 v1.0.22 才能拿到。
- `find-process-mode` 默认调整只在 Windows 生效；macOS / Linux 仍为 always。
