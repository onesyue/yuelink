## Bug Fixes

- **Android 蜂窝下连接不稳定** — 修复 v1.0.23 引入的回归。心跳从固定 10/60 s 改成 4-tier 后(15/30/60/120 s)，蜂窝前台需要 5 × 30 s = 150 s 才触发 silent restart，连接断了 2.5 分钟才自动恢复。这版改两件：失败阈值从 5 收紧到 3；首次心跳失败立即把下次 tick 切到 5 s（self-rescheduling），失败计数归零再回原 cadence。蜂窝下从一次失败到 silent restart ≤ 15 s，比 v1.0.22 还快；健康路径电池友好的 30 s/120 s 不变。
- **中国境内打开「签到日历」一直转圈圈** — v1.0.24 的 `calendar_page` 和 `resign_dialog` 直接 new 出 CheckinRepository 没传 mihomo proxy port，HTTP 走 DIRECT，`yue.yuebao.website` 在中国被拦死。改为通过 `checkinRepositoryProvider` 拿 mihomo-proxy-aware 实例，与签到 / 补签的 POST 一致经 VPN 出口出去。已签到正常 + 日历转圈 = 这个回归的指纹。

## Upgrade Notes

- 已登录用户无需重登，订阅与设置自动保留。
- 蜂窝网络下连接稳定性必须升级到 1.0.25 才能拿到（修复全在客户端心跳逻辑）。
- 中国境内用户的签到日历修复同理。
- iOS / Android / macOS / Windows / Linux：直接覆盖 1.0.24。
