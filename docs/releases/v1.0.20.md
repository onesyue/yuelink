## Performance

- **mihomo 吞吐回归上游默认(约 +30%)**:配置模板不再注入 `parse-pure-ip` / `force-dns-mapping`。这两项在日常网络上各自约吃掉 20–30% 的吞吐。订阅里显式打开的设置保持不动。

## What's New

- **主页模式徽章可点击切换**:「规则 / 全局 / 直连」徽章点一下轮转切换,「TUN / 系统代理」徽章点一下热切换(仅桌面)。状态变化带 ripple 反馈并持久化,不用再绕到节点页或设置页。

## Bug Fixes

- **Windows 托盘退出卡死兜底**:连接状态下从托盘右键退出,如果 `exit(0)` 被某个 native 回调阻塞,3 秒硬兜底 `Process.killPid` 发送 `TerminateProcess`,不再出现「窗口关了但任务管理器里还在」。
- **前后台恢复收紧**:切回前台不再被动翻回连接状态,尊重用户手动停止;resume 处理器串行化、WebSocket 增加空闲看门狗、运营商启动去重。
- **「我的」页账号信息不再永久卡加载中**:向 `yue.yuebao.website` 的请求加 request / body 级别 10 秒超时,后端接了请求但不回包时现在最迟 10 秒兜底回退。
- **节点页分组标题右对齐**:「选中节点 / 节点数量 / 闪电」三件套在所有分组都贴齐右边,不再留视觉空隙。
- **代理保护**:三方代理劫持 30 秒内响应恢复,对短暂写入放宽容忍;Linux 不可验证情况下跳过破坏性 tamper restore。
- **测速**:单组节点全部超时后,重连成功即自动清空超时缓存。
- **Riverpod 3 生命周期**:post-dispose state write 在 5 个 notifier 上全部守护,macOS 集成测试里的 `UnmountedRefException` 不再复现。
- **runtime 诊断**:4 处 silent catch 按语义分流到 `ErrorLogger`(真 bug)或 `EventLog`(预期流控)。
- **更新器**:legacy check 失败只在手动检查时报给用户。
- **geodata**:CI 产物按 SHA-256 严校验;运行时镜像下载 fail-soft,CI 源在 jsdelivr 失步期间切到 `raw.githubusercontent.com`。
- **构建工具链**:Android AGP 8.7.3 → 8.9.1、Gradle 8.9 → 8.11.1;CI 加 `GOFLAGS=-mod=mod` 以接受 mihomo v1.19.24;iOS 部署目标与 Podfile 底线对齐。
