正式版，发布日期 2026-04-20。基于 v1.0.19 之后 74 个 commit，覆盖工程治理、稳定性与若干用户面修复。`v1.0.20-pre` 已在内部小范围验证，本次发正式版。

### 发布亮点

- **mihomo 吞吐回归上游默认（约 +30%）**：配置模板不再强制注入 `parse-pure-ip` 和 `force-dns-mapping`，回到 mihomo 原厂默认值。这两项在日常网络上各自约吃掉 20–30% 的吞吐。订阅里显式打开这两项设置保持不动。(`85a316e`)
- **主页模式徽章可直接点击切换**：Dashboard 的「规则 / 全局 / 直连」徽章点一下轮转切换，「TUN / 系统代理」徽章点一下热切换（仅桌面）。带 ripple 反馈，状态变化立即在徽章上体现，并持久化到下次启动。不用再绕到节点页或设置页。(`9800093`、`cde60c1`)
- **Windows 退出卡死兜底**：连接状态下从托盘右键退出，如果 `exit(0)` 被某个 native 回调阻塞，3 秒硬兜底会调用 `Process.killPid` 发送 `TerminateProcess`，确保进程彻底结束——不再出现「窗口关了但任务管理器里还在」的症状。(`ea493e9`)
- **前后台恢复收紧**：切回前台不再「被动翻回连接状态」，尊重用户的手动停止操作；resume 处理器串行化、WebSocket 增加空闲看门狗、运营商启动去重。(`673b966`、`def6b5a`)
- **「我的」页账号信息不再永久卡加载中**：通过 YueLinkHttpClient 向 `yue.yuebao.website` 的请求加了 request/body 级别的 10 秒超时。之前只有 TCP 握手超时守护，后端接了请求但不回包时 `_ProfileRow` 会永久显示「加载中」，下拉刷新 / resume / 一键修复都没法自愈。现在最迟 10 秒兜底回退，用户能手动重试。(`071fc85`)
- **节点页分组标题右对齐恢复**：之前「选中节点 / 节点数量 / 闪电」三件套在某些分组上会被 `Flexible` 的 loose fit 留出视觉空隙、看起来不贴右边框；改为 `ConstrainedBox` 不参与 Row flex 分配，三件套在所有分组都贴齐右边。(`5d0ac22`)

### 修复

- **代理保护**：三方代理劫持 30 秒内响应恢复，对短暂写入放宽容忍；Linux 不可验证情况下跳过破坏性的 tamper restore。(`63aaba8`、`f9fad31`)
- **测速**：单组节点全部超时后，重连成功即自动清空超时缓存，不再要求用户手动刷新。(`910e9ea`)
- **Riverpod 3 生命周期**:`post-dispose state write` 在 5 个 notifier 上全部守护，macOS 集成测试里的 `UnmountedRefException` 不再复现。(`13e9d12`)
- **runtime 诊断**:4 处 silent catch 按实际语义分流到 `ErrorLogger`（真 bug）或 `EventLog`（预期流控），关键路径失败不再默默吞掉。(`6c7f9d0`、`4a56b1a`)
- **更新器**:legacy check 失败只在手动检查时报给用户，后台自动检查不再弹不必要的错误。(`9d9b184`)
- **geodata**:CI 产物按 SHA-256 严校验；运行时镜像下载 fail-soft，CI 源在 jsdelivr 失步期间切到 `raw.githubusercontent.com` 保证发布可靠。(`84ce9a1`、`f9e1aa7`)
- **构建工具链**:Android AGP 8.7.3 → 8.9.1、Gradle 8.9 → 8.11.1；CI 加 `GOFLAGS=-mod=mod` 以便 Go 1.26.2 接受 mihomo v1.19.24 的 go.mod；iOS 部署目标与 Podfile 底线对齐。(`80c53c9`、`c399832`、`214f3c6`)
- **产物命名规范化**:`*-pre` 版本化 tag 触发的 CI 构建现在正确附加 `-pre` 后缀，避免预发布产物和正式版产物名冲突。(`64e0dad`)

### 内部工程改进

这一版的大部分精力放在工程面。对日常使用没有可见变化，但后续维护与扩展更顺：

- **Store 三层重构收口**:`lib/{domain,infrastructure,modules}/store/*` 完成 S1-S6 迁移。`PurchaseState` 进 domain、`CheckoutResult` 进 infrastructure、`PurchaseNotifier` 独立拆出、订单结果 UI 拆分、XBoard 异常统一走 `StoreError`、真环境回归 harness 就位。
- **Settings 页拆分**:General Settings 页按 Appearance / Updates / Privacy / ServiceMode / Hotkey / CloseBehavior / SplitTunnel 等 section 全部抽成独立 widget。
- **Core 三管理器解耦**:`CoreLifecycleManager` / `CoreHeartbeatManager` / `SubscriptionSyncService` 不再直接依赖 `proxyGroupsProvider` / `profilesProvider`。
- **诊断与测试**:`ProfilesNotifier` dispose guard 回归测试新增;`app_notifier._currentTopTimer` 死字段清理;analyze info 从 105 → 14。
- **依赖维护升级（非 win32 链）**:`flutter_riverpod 2.6.1 → 3.3.1`、`fl_chart 1.2.0`、`app_links 7.0.0`、`mobile_scanner 7.2.0`、`slang 4.14.0`、`google_fonts 8.0.2`、`launch_at_startup 0.5.1`、`tray_manager 0.5.2`、`window_manager 0.5.1`、`flutter_lints 6.0.0`。
- **mihomo**:v1.19.23 → v1.19.24（上游最新稳定）,setup 脚本自动 apply `core/patches/`。
- **CI 固化**:Flutter 3.41.7 全工作流对齐。
- **未来准备**:`SecureStorageMigration` backup skeleton + 6 cases 测试到位,为将来的 `flutter_secure_storage 9 → 10` 升级铺路;win32 5 → 6 升级约束链写进 `docs/dependency-upgrade-plan-win32-v6.md`。**这些是内部准备,没有在启动路径被启用**。

### 校验说明

- 每个正式产物都会附带一个同名 `.sha256` 文件，用于校验下载完整性。
- Android 用户优先下载 `android-universal.apk`；只有明确知道设备架构时，才建议改下分 ABI 包。
- Windows 默认推荐 `setup.exe`；只有不方便安装或需要绿色版时，再使用 `portable.zip`。
- iOS 包为侧载用途，不走 App Store 分发。

本次正式版完成版本号、tag、产物命名三者对齐：

### 已知问题

- **Analyze 剩余 14 条 info**，全部在红线 / 历史不动区（`main.dart` / `nodes_page.dart` / `service_manager.dart` / `test/manual/`），均为纯风格 lint，无功能影响。
