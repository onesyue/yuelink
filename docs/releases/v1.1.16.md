## Fixes

- 桌面 TUN ↔ 系统代理切换不再卡在"DNS 接管失败"误报。状态判定改为以实际运行的核心为准，不再被界面上的"目标模式"骗到。
- 同时跑 WireGuard / Tailscale / Cisco AnyConnect 等其他 VPN 时，TUN 路由检查只认走全局默认路由的 utun，不再因为机器上有别的 utun 接口就直接判定"TUN 已生效"或"清理失败"。
- DNS 接管检测改端到端：发起一次系统级 DNS 解析、检查返回是否落在 fake-IP 段（198.18.0.0/15），更贴近真实流量路径，不再依赖 mihomo 内部探针在节点波动时假阴性。
- macOS TUN 上次会话异常退出导致系统 DNS 被遗留为公网解析器（114 / 223 / 8.8.8.8）时，下次正常停止识别为残留并改回 DHCP 自动获取，不再永久污染原始 DNS 设置。
- macOS TUN 同时接管 IPv6：原本 IPv6 流量绕过 TUN 直接走系统默认 v6 路由会泄漏真实 IPv6 地址，现在 TUN 接口同时分配私有 ULA 并接管 v6 全局路由。
- 用户"断开"后，心跳自动恢复 / 网络切换重连不再反向覆盖断开操作。
- 切换 TUN ↔ 系统代理过程中，停止那一步不再写入 manual-stop 标记。
- 桌面服务模式启动失败时即使诊断报告写入异常，状态标记也保证复位；helper service 显式停止时清零 watchdog 崩溃计数。

## Upgrade Notes

- 已登录用户无需重登。登录态、订阅、Emby、播放记录、本地设置全部保留。
- macOS 用户：之前 v1.1.15 或更早版本异常退出过的，本版第一次正常停止 TUN 时会自动把残留的公网 DNS 改为 DHCP 自动获取。如果你本来就手动配的这个组合，停止后请到 系统设置 → 网络 → DNS 重新填回。
- macOS 桌面 TUN 现在接管 IPv6 全局流量。代理节点不支持 IPv6 出向时，访问纯 IPv6 站点（少数 Cloudflare 边缘、部分企业网）可能比之前慢一拍（连接失败后回退 IPv4）；大部分双栈站点无感。
