## Fixes

- DNS 不再泄漏到国内 DoH。之前 TUN 模式下访问国外网站时 DNS 查询会被 AliDNS / TencentDNS 看到。订阅模板对非中国域名走 Cloudflare DoH / Google DoH；客户端在订阅未带兜底规则时主动注入相同策略。fake-ip-filter 末尾加 geosite:cn，让整个中国 geosite 数据集（约 10 万域名）走真实 DNS 解析，国内 CDN 调度更准、首屏更快。
- 死节点自动跳过。11 个国家聚合（HK / TW / JP / KR / SG / US / UK / DE / NL / AU / CA）从负载均衡改为延迟竞速 + HTTP 200 健康检查；TLS 握手失败 / 服务端 reset / Cloudflare 反代 403 都算"不健康"，节点会被自动踢出聚合，下次选最快的活节点。
- 流媒体偶发被错路由到地区组：sniffer 强制嗅探列表新增 Netflix / Disney+ / AWS / DSS 系列 CDN 域名，按真实业务域名路由到流媒体组，不再因为 IP 归属国家被分到地区聚合。
- TCP 长连接稳定性优化：之前 keep-alive interval 配 30s 但 idle 没设，落到操作系统默认 7200s 才发第一个探测包，等于配置形同虚设，挂掉的 NAT 绑定被发现得晚（用户体感是"用着用着突然卡 1 分钟"）。新加 keep-alive idle 600s，匹配 Clash Verge Rev / mihomo-party 主流默认。
- Apple Push / iCloud 等 fake-IP 长连接重连提速：开启 fake-IP 表跨重启持久化，重启后操作系统 DNS 缓存里的 fake-IP 还能正确反查到原域名，前几秒首包延迟消除。

## Upgrade Notes

- 拉新订阅一次即生效。已登录用户无需重登，订阅、设置、节点选择全部保留。
- AI 策略组的当前选择保留。如果之前给 AI 组选了某个普通国家聚合（如台湾聚合 / 日本聚合），cf-fronted 服务（ChatGPT / Claude）可能仍被 Cloudflare WAF 拒绝（这些聚合里装的是普通 IDC 节点，出口 IP 在 cf 黑名单）。建议手动把 AI 组切到具体的 AI 解锁专用节点：在 yuelink UI 的 AI 组下拉，选 Yue.to 悦通，找名字含"美国 AI 解锁 1"/"日本 AI 解锁 1"/"新加坡 AI 解锁 1"的节点。
- 没有改 mihomo 内核版本、TUN 配置、节点选择口径。
- 已知遗留：同步订阅完成后没自动调用 mihomo 配置重载，新写的 yuelink-service.yaml 要等下次 mihomo 自然重载或重启 yuelink 才生效。下版会修。
