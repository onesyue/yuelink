## Fixes

- **支付失败可解释** — 购买失败现在会区分登录过期、支付链接缺失、支付页打不开、订单取消、网络异常和支付通道拒绝，并在失败页给出对应恢复动作。
- **支付失败 telemetry 补全** — `purchase_fail` 增加 `failure_kind` 字段，方便在 telemetry dashboard 直接定位支付失败主因。

## Improvements

- **续费提醒闭环** — 首页在订阅 T-7 / T-3 / T-0 / 已过期窗口展示续费提醒卡，点击直达套餐中心，并记录 `renewal_banner_show` / `renewal_click` / `renewal_pay_success`。
- **诊断包入口前置** — 连接修复页主按钮下方新增“导出诊断包”入口，失败现场不必滚到页面底部再找日志导出。
- **连接修复动作埋点** — 一键修复、重启核心、重建 VPN、同步订阅、清缓存、电池白名单等动作会记录 attempt/result，便于判断哪类自助修复真正有效。
- **Telemetry 连接健康面板** — dashboard 新增 Connection MTTI inputs 卡片，聚合 `connect_failed`、启动失败、诊断包导出和修复动作成功率，帮助 30 分钟内定位平台、阶段和修复方向。
