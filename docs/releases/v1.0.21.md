# YueLink v1.0.21

v1.0.21 是 v1.0.20 之后的窄范围 hotfix，专门收口 v1.0.20 用户反馈中复现率最高的几个问题。基于 v1.0.20（7b2bed8）之后 9 个 commit，无功能新增，无依赖升级。

## Performance

mihomo 吞吐对齐上游默认：客户端 fallback 配置与服务端订阅模板都不再注入 `parse-pure-ip` / `force-dns-mapping`。这两项在日常网络上会明显影响吞吐；订阅里显式打开的设置保持不动。

服务端 XBoard 订阅模板 `/home/xboard/yue-to/rules/{default,app}.clash.yaml` 已同步对齐：删除 `parse-pure-ip: true`，并将 `store-fake-ip: true` 修正为 `store-fake-ip: false`。下次同步订阅后自动生效，无需用户手动处理。

## What's New

托盘菜单支持模式快切：桌面端右键托盘图标后，可在「模式」里快速切换规则 / 全局 / 直连 / 系统代理 / TUN，并用勾选状态标记当前模式。

自动恢复连接增加透明提示：心跳连续失败触发自动恢复时，现在会显示「正在自动恢复连接...」「已自动恢复连接」或「自动恢复失败，连接已断开」，不再静默闪烁。恢复过程也会写入 EventLog，方便后续导出诊断。

桌面 TUN / 系统代理切换更可靠：切换 TUN 时改走完整 service helper restart 流程，失败会回滚到原模式并尝试恢复，避免 UI 显示已切换但 runtime 实际没有切过去。

## Bug Fixes

断开后切回不再误显示「已连接」：用户在 Android 上主动断开后，切到其他应用再切回 YueLink，不再出现 UI 显示节点名和连接图标、但实际无网的状态。

用户主动断开状态持久化：`userStoppedProvider` 不再只存在内存中。Android 后台杀引擎、冷启动、resume 都会先读取持久化标记，避免把用户主动断开的状态误恢复成 running。

开启 auto-connect 的用户，断开后杀 app 重开也不会被误启动；但用户手动 connect 后，切应用回来仍然正常工作。

v2rayN / vr2 等代理客户端劫持系统代理后能立即恢复：system proxy 校验增加强制刷新路径，`set()` / `clear()` 后立即清缓存，resume 时如果处于 systemProxy 模式，会直接强校验并按需 restore，不再最长等待 60 秒缓存 + 30 秒心跳。

断开重连后立即测速不再“全红”：`testGroupDelay()` 抛异常时也会走 flush + retry recovery，不再把所有节点直接标红。telemetry 增加 `reason` 字段，用于区分 `all_timeout` 和 `exception`。

Windows 托盘退出彻底：3 秒退出兜底 watchdog 移到独立 isolate。即使主 isolate 被 native 调用卡住，也能触发 SIGKILL / TerminateProcess，不再出现窗口关闭但 `yuelink.exe` 仍留在任务管理器里的情况。

路由模式切换恢复 verify 校验：`ModeActions.setRoutingMode()` PATCH 成功后会再读取一次 mihomo 实际模式。如果订阅强行覆盖请求，会给出 warning 提示，而不是无声 success。

安装 / 更新服务模式更稳：helper readiness 最长等待 12 秒，并增加 3 次启动重试。安装或更新服务会被视为用户想启用 TUN，会自动清掉 manualStopped 标记，避免服务装好后不自动启动。

## Upgrade Notes

已登录用户无需重登，订阅与设置自动保留。

吞吐与 `cache.db` 相关修复通过服务端模板对齐生效，下次同步订阅自动拿到。此项对客户端版本本身无强制升级要求，但只有 v1.0.21+ 才同时包含本次客户端侧修复。

断开 / 自动恢复 / 系统代理校验类修复 100% 客户端侧，必须升级到 v1.0.21 才能拿到。

Windows 退出 / 桌面 TUN 切换类修复同样需要升级到 v1.0.21。

Android 用户优先下载 `android-universal.apk`；只有明确知道设备架构时，才建议下载分 ABI 包。

Windows 默认推荐 `setup.exe`；只有不方便安装或需要绿色版时，再使用 `portable.zip`。

iOS 包为侧载用途，不走 App Store 分发。
