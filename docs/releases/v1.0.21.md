## What's New

- **托盘菜单新增「模式」快切** — 规则 / 全局 / 直连 / 系统代理 / TUN 在托盘右键直接切换,状态与主页同步,不需要再打开主窗口。
- **自动恢复连接有了过程提示** — 心跳失败触发的静默重启不再悄悄发生,依次提示「正在自动恢复连接」/「已自动恢复连接」/「自动恢复失败」,全过程落在事件日志里。

## Bug Fixes

- **Android 断开后切回不再误显示「已连接」** — 「用户主动停止」状态持久化到磁盘,后台被杀掉重建引擎后也能正确恢复。
- **系统代理被 v2rayN / Clash Verge 等第三方劫持后能立即恢复** — resume 与窗口聚焦时强制重新校验,不再等 60 秒缓存过期。
- **断开重连后测速不再「全红」** — `testGroupDelay` 抛异常的路径现在和全超时路径走同一套 flush + retry。
- **mihomo 吞吐对齐上游默认(约 +30%)** — 客户端和服务端模板同步清理 `parse-pure-ip` / `force-dns-mapping` 两个残留 flag。
- **Windows 托盘退出更彻底** — 3 秒退出兜底移到独立 isolate,主 isolate 卡死也能 `TerminateProcess`。
- **桌面 TUN / 系统代理模式切换现在真的切换了** — 从 `PATCH /configs` 改为完整 restart,失败时回滚到原模式。
- **路由模式切换恢复校验** — 切换后会比对 mihomo 实际模式,订阅强制覆盖时会提示而不是静默成功。

## Server-side

XBoard 订阅模板已同步对齐,所有使用该面板的客户端下次同步订阅即自动生效,无需操作:
- 删除 `parse-pure-ip: true`
- `store-fake-ip: true` → `false`

## Upgrade Notes

- 已登录用户无需重登,订阅与设置自动保留。
- 吞吐与缓存冲突修复走服务端模板,任意客户端版本下次同步订阅都会拿到。
- 其余修复(断开识别 / 自动恢复 / 系统代理校验 / Windows 退出 / TUN 切换)需升级到 1.0.21。
