## What's improved

- **设计系统统一** — 订阅、套餐、订单、签到、偏好、配置覆写、连接修复、规则模块、公告等 12 个二级页面迁移到统一的 inset-grouped + 大标题 scaffold，桌面侧自动收紧最大宽度，不再撑满整屏。Material icons 全部走 rounded 变体。
- **iOS 26 风格底栏** — 玻璃材质 + 切换图标动画的底部导航；Android 端避开了带玻璃模糊的 ANR 路径，切 tab 不再卡顿。
- **macOS 任务栏图标随连接状态切色** — 已连接 / 未连接一眼可分；之前需要打开主窗口才能确认状态。
- **桌面 TUN 诊断更可信** — controller、TUN 网卡、路由、DNS、基础连通性五项全部通过才显示"已连接"；缺一项就给出具体原因（驱动缺失 / 权限不足 / 路由未接管 / DNS 接管失败 / cleanup 残留）。
- **Wintun 打包闭环** — Windows 安装包从上游 wintun.net 下载并 sha256 校验后再打包；缺驱动时 UI 显示"Wintun 驱动缺失"，不再"假已连接"。
- **服务端节点治理基建** — 后端开始按 RUM + 主动探测的双源数据为每个节点打状态标签；超时多 / 普通站正常但 Claude/ChatGPT 403 等场景能被独立识别。任何下线动作都需要人工确认，不会自动把节点踢出默认组。
- **凭据保护** — 新增提交前的敏感字段扫描器（数据库 DSN、订阅 token、Reality 公钥 / shortId 等），CI release-gate 阻断含敏感字段的发布。

## Upgrade Notes

- 已登录用户无需重登，订阅与设置自动保留。
- 默认遥测保持关闭。开启位置：设置 → 隐私 → 匿名遥测。
- 直接覆盖安装即可，所有平台。
- Windows 升级会自动带上 wintun.dll；如果之前提示"Wintun 驱动缺失"，本版本应该自愈。
- 本版本是 UI 现代化 + 桌面 TUN 诊断闭环 + 服务端节点治理基建版本。**节点池稳定性、AI 出口池可用性、桌面 TUN 全平台长周期稳定性仍在治理中**，不应解读为节点、AI 或 TUN 的稳定性治理已经完成。
