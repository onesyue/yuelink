## New

- **跟随系统语言** — 装好 yuelink 默认按手机 / 电脑系统语言显示中文或英文，不再硬编码中文；之后用户在系统设置切换语言（Android 设置 → 语言、iOS 设置 → 通用 → 语言、macOS 系统设置 → 通用 → 语言、Windows 设置 → 时间和语言 → 语言）yuelink **实时跟着切**，不需要重启 App、不需要进设置改。设置 → 通用 → 语言 三选：跟随系统 / 中文 / English。

- **AI 节点缺位时仍能解 DoH** — 用户订阅没有 AI 解锁聚合组、或者 AI 组里全是被 Cloudflare WAF 黑名单的普通节点时，Chrome SecureDNS / DoH 整套（`cloudflare-dns.com` / `chrome.cloudflare-dns.com` / `mozilla.cloudflare-dns.com` / `dns.google` / `cloudflare-ech.com`）会失败，连带 google.com / github.com 都打不开。本版本当订阅同时含 AI 组和主组时，自动注入合成 `_YueLink_SecureDNS` fallback 组（`type: fallback, proxies: [<AI>, <主组>], url: gstatic/generate_204, interval: 300`）。AI 节点全死时 mihomo 健康检查自动退到主组 —— 至少非 cf 服务恢复正常解析。cf-fronted 的 ChatGPT / Claude 仍需服务端模板有 cf-friendly 节点。

- **代理互斥提示（Android）** — 启动 yuelink 前先探测系统是否已有别的 VPN 在跑（`ConnectivityManager.allNetworks` + `TRANSPORT_VPN`），命中时弹"另一 VPN 在运行"对话框 + 一键跳"系统 VPN 设置"。Android 系统每次只允许一个 VPN，之前没这个预检会让用户卡在转圈不知道哪里有问题。设计参考 Clash Meta / Verge Rev 的 mutex 流程。iOS 单 VPN 槽 OS 自带切换提示，Desktop TUN ↔ SystemProxy 是 connectionMode 单选 enum 天然互斥，三平台覆盖完整。

- **桌面 TUN 切到 jumbo 虚拟 MTU（9000）** — 与 mihomo Meta 主线一致；userspace stack 仍按物理接口 MTU 分段，但虚拟帧变大显著降低 TUN 读写次数 / 上下文切换开销。Android 早就是 9000，iOS 仍 1500（cellular socket 限制）。

- **Android 局域网精确放行（仿 Clash Meta 路由表）** — Android VpnService 不再用 `0.0.0.0/0` 钝路由把 LAN / hotspot / 路由器管理页一并拉进 TUN，改为按 Clash Meta 的 PUBLIC_IPV4_ROUTES 拆分覆盖公网段，自动绕开 `10/8` `127/8` `169.254/16` `172.16/12` `192.168/16` `224/4`。Android Q+ 同时通过 `setHttpProxy` + 排除列表对私网服务直连。和 v1.1.21 已经做的 iOS `excludedRoutes` / Windows LAN compat 路线对齐。

## Fixes

- **同步订阅后立即生效** — `syncSubscription` 写完 profile yaml 后调 `coreActions.restart(yaml)` 重新加载 mihomo runtime config。v1.1.20 release notes 里的"下版会修"在 v1.1.22 落地。

- **Android TUN 重连后选择器 / fake-IP 卡死** — 用户手动 rule→global→rule 切一遍能恢复的那个老 bug：mihomo Android TUN 重连后偶尔会留 stale fake-IP / selector 状态。每次启动强制 `setRoutingMode(savedMode)`（即使保存值就是默认 `rule`），给 mihomo 一次明确 nudge。

- **系统语言切换 yuelink 不跟随** — 之前 stored language 一旦写过就永远不再 resolve 系统 locale。新增 `WidgetsBindingObserver.didChangeLocales` 钩子，当用户偏好是 `auto` 时实时切到系统当前语言；`S.setLanguage(next)` 同步更新 slang 全局 `t`，让 tray menu / notification 等不在 widget tree 里的字符串也跟着切。

## Internal

- **`lib/i18n/locale_resolver.dart`** — `LanguagePreference` 常量 + `currentSystemLanguage()` + `effectiveLanguageForPreference()` 单一入口，bootstrap / settings UI / `didChangeLocales` 三处共用同一解析逻辑，杜绝 preference → rendered locale 漂移。`PlatformDispatcher.instance.locale` 优先于 `Platform.localeName`（早 bootstrap 阶段更可靠）。

- **`LanguagePreferenceNotifier` 与 `LanguageNotifier` 拆分** — preference (auto/zh/en) 是设置层，render (zh/en) 是应用层。MaterialApp.locale 只读 render；settings UI 只写 preference + 立即派生 render；didChangeLocales 仅在 preference == auto 时同步 render。

- **`_YueLink_SecureDNS` 注入器**（`rules_transformer.dart::_ensureSecureDnsFallbackGroup`） — proxy-groups 末尾追加 fallback 组；name / proxy entries 双引号包裹支持 emoji / CJK 组名；`_isSafeRuleTarget` 上游过滤掉含 `,` / `"` 的诡异组名。单组（仅 AI 或仅主）退化为 v1.1.21 单点路径，旧契约保留 + 1 项新单测。

- **Android `MainActivity::checkVpnConflict`** MethodChannel handler — `ConnectivityManager.allNetworks` + `NetworkCapabilities.hasTransport(TRANSPORT_VPN)` 扫描；自身 `serviceBound + getTunFd > 0` 排除自家 VPN，避免重连场景误报为外部冲突。`openVpnSettings` 失败降级到 `ACTION_SETTINGS`。`NetworkCapabilities.getOwnerUid` 是 `NETWORK_SETTINGS` 特权方法，普通 App 拿不到，所以对话框只能说"另一 VPN"不指名。
