## New

- 跟随系统语言：装好默认按手机 / 电脑系统语言显示中文或英文。系统设置切换语言后 yuelink 实时跟着切，不需要重启 App、不需要进设置改。设置 → 通用 → 语言 三选：跟随系统 / 中文 / English。
- AI 节点缺位时仍能解析 DoH：订阅没有 AI 解锁组、或 AI 组里全是被 Cloudflare WAF 黑名单的普通节点时，Chrome SecureDNS / DoH 会失败，连带 google.com / github.com 都打不开。本版在 AI 组与主组同时存在时自动注入合成的 SecureDNS 兜底组，AI 节点全死时自动退到主组，至少非 cf 服务能正常解析。cf-fronted 的 ChatGPT / Claude 仍需服务端模板有 cf-friendly 节点。
- Android 代理互斥提示：启动前先探测系统是否已有别的 VPN 在跑，命中时弹"另一 VPN 在运行"对话框 + 一键跳系统 VPN 设置。Android 系统每次只允许一个 VPN，之前没这个预检会让用户卡在转圈不知道哪里有问题。
- 桌面 TUN 切到 jumbo 虚拟 MTU（9000），与 mihomo Meta 主线一致；Android 早就是 9000，iOS 仍 1500。
- Android 局域网精确放行：VpnService 不再用 0.0.0.0/0 钝路由把局域网 / 路由器管理页 / 打印机 / hotspot 一并拉进 TUN，改为按公网段拆分覆盖，自动绕开 10/8、127/8、169.254/16、172.16/12、192.168/16、224/4。

## Fixes

- 同步订阅后立即生效：之前要等下次 mihomo 自然重载或重启 yuelink 才生效（v1.1.20 已知遗留），本版同步完成后立即重新加载 mihomo 运行配置。
- Android TUN 重连后选择器 / fake-IP 偶发卡死：用户手动 rule → global → rule 切一遍能恢复的那个老问题，每次启动强制重新应用一次路由模式给 mihomo 一个明确 nudge。
- 系统切换语言 yuelink 不跟随：之前存储的语言一旦写过就永远不再 resolve 系统 locale。现在偏好为"跟随系统"时实时跟着切。

## Changed

- 撤回 v1.1.21 的几处 UI 入口，回归 v1.1.20 视觉：Dashboard 顶部 Private DNS 提示横幅、设置 → 高级里的"导出诊断报告 / 复制 Windows 诊断脚本 / 局域网兼容模式 / 托盘空闲标志"四行。底层服务保留，未来按需重新接入 UI。
- 删除 v1.1.21 引入的 Dashboard 连通性体检卡。

## Upgrade Notes

- 拉新订阅一次即生效。已登录用户无需重登，订阅、设置、节点选择全部保留。
- 老用户的"中文"是显式偏好，会被尊重，升级后 yuelink 仍是中文。要享受跟随系统请进 设置 → 通用 → 语言 → 跟随系统。新装用户默认即跟随。
- 没有改 mihomo 内核版本、TUN bypass 列表、节点选择口径。
