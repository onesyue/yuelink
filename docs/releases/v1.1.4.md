## What's Improved

- **全局 UI 正式收敛** — 订阅、套餐、订单、签到、偏好设置、配置覆写、连接修复、规则模块、公告等二级页面统一为 compact inset-grouped 结构，去掉重复大标题和重影，桌面端自动限制内容宽度。
- **轻玻璃视觉系统** — 底部四个 tab 使用轻量玻璃浮层；主要页面背景、列表区块、设置卡片、订阅卡、套餐卡、节点组、签到卡、公告卡和规则模块卡片统一为半透明玻璃表面。滚动卡片不使用重 blur，避免 Android / Windows / Linux 滚动性能抖动。
- **诊断导出更可靠** — 修复中文 Windows 下 `netsh` / `ipconfig` 输出乱码；修复日志导出遇到非 UTF-8 字节时失败的问题，诊断包会尽量保留可读内容而不是直接失败。
- **连接修复归因更清楚** — 连接修复页继续区分 controller、TUN 网卡、路由、DNS、系统代理冲突、节点 timeout、AI 出口受限和 Reality 认证失败，不再把所有问题都写成“节点不可用”。
- **桌面 TUN 诊断闭环** — controller、TUN 网卡、路由、DNS、基础连通性通过后才允许显示已验证状态；缺驱动、缺权限、路由未接管、DNS 接管失败、cleanup 残留会显示具体原因。
- **Wintun 打包门禁** — Windows 构建流程会下载并校验上游 Wintun，构建后检查 `wintun.dll` 是否进入产物；缺失时阻断发布。
- **节点治理基础设施** — telemetry、RUM 聚合、active probe 骨架、P7 节点状态机和 review queue 已进入正式版本。节点下线仍需人工确认，不会自动踢出默认组。
- **发布安全门禁** — release-gate 会检查 telemetry 路由、server 聚合测试、Wintun hash 文件和敏感字段扫描，降低“路由 404 / 凭据误入仓库 / Windows TUN 缺驱动”进入正式产物的风险。

## Upgrade Notes

- 已登录用户无需重登，订阅与设置自动保留。
- 默认匿名遥测仍保持关闭；开启位置：设置 → 隐私 → 匿名遥测。
- 直接覆盖安装即可，所有平台。
- 如果打开代理后仍显示“连接失败”，请进入 设置 → 连接修复，先看具体归因，再导出诊断包。
- Windows 如果仍提示“Wintun 驱动缺失”，请使用正式安装包重新安装，并导出诊断包反馈。

## Known Limits

- 本版本可以作为 **正式版 UI / 诊断 / 遥测治理版本** 发布。
- 节点池稳定性仍在治理中，本版本不承诺默认组质量已经完成治理。
- Claude / ChatGPT 出口风控仍可能存在，本版本不承诺 AI 出口池已经完成治理。
- 桌面 TUN 已修复假开启和诊断缺失问题，但 Windows / macOS / Linux 全矩阵与 24h soak 仍需继续验证，本版本不承诺 TUN 长周期稳定性已经完成治理。
