# YueLink v1.0.20-pre

预发布候选，日期待定。基于 v1.0.19 之后 70 个 commit 的打磨版本。大部分改动集中在稳定性、吞吐、构建与架构整理，仅少量用户可感知的交互改动。

## 发布亮点

- **mihomo 吞吐回归上游默认（约 +30%）**：配置模板不再强行开启 `parse-pure-ip` 和 `force-dns-mapping`，回到 mihomo 原厂默认。这两项在日常网络上各自约吃掉 20–30% 的吞吐，订阅里显式打开的设置保持不动。(`85a316e`)
- **主页模式徽章可以直接点击切换**：Dashboard 上的「规则 / 全局 / 直连」徽章现在点一下就轮转切换；「TUN / 系统代理」徽章点一下热切换（仅桌面）。之前只是显示用的徽章现在带 ripple 反馈，不用再绕到节点页或设置页。(`9800093`)
- **Windows 退出卡死兜底**：在 Windows 下，右键托盘退出时，如果 `exit(0)` 被某个 native 回调堵住（见过 `windowManager.destroy` / `trayManager.destroy` / 服务 helper IPC 等路径），3 秒硬兜底会 `Process.killPid` 直发 `TerminateProcess`，确保进程一定退出，不再出现「窗口关了但任务管理器里还在」的症状。(`ea493e9`)
- **app 前后台恢复行为收紧**：切回前台后不再「被动翻回连接状态」，尊重用户手动停止；resume 处理器串行化、WebSocket 增加空闲看门狗、运营商启动去重。(`673b966`、`def6b5a`)

## 修复

- **代理保护**：三方代理劫持后 30 秒内响应恢复，对短暂写入放宽容忍；Linux 不可验证情况下跳过破坏性的 tamper restore。(`63aaba8`、`f9fad31`)
- **测速**：单组节点全部超时后，重连成功即自动清空超时缓存，不再要求用户手动刷新。(`910e9ea`)
- **Riverpod 3 生命周期**：`post-dispose state write` 在 5 个 notifier 上全部守护，macOS 集成测试里的 `UnmountedRefException` 不再复现。(`13e9d12`)
- **runtime 诊断**：4 处 silent catch 按实际语义分流到 `ErrorLogger`（真 bug）或 `EventLog`（预期流控），关键路径失败不再默默吞掉。(`6c7f9d0`、`4a56b1a`)
- **更新器**：legacy check 失败只在手动检查时报给用户，后台自动检查不再弹不必要的错误。(`9d9b184`)
- **geodata**：CI 产物按 SHA-256 严校验，运行时镜像下载 fail-soft。(`84ce9a1`)
- **iOS 部署目标**：和 Podfile 底线对齐，避免构建期警告。(`214f3c6`)
- **Android Gradle**：AGP 8.7.3 → 8.9.1，Gradle 8.9 → 8.11.1，配合 Flutter 3.41.7 CI 环境。(`80c53c9`)

## 内部工程改进

这轮的大部分精力放在工程面。对日常使用没有可见变化，但后续维护与扩展会更顺：

- **Store 三层重构收口**：`lib/{domain,infrastructure,modules}/store/*` 完成 S1–S6 迁移。`PurchaseState` 进 domain、`CheckoutResult` 进 infra、`PurchaseNotifier` 独立拆出、订单结果 UI 拆分、XBoard 异常统一走 `StoreError`、真环境回归 harness 就位。(`bf86e2b`、`612f76e`、`2e873f5`、`ca34c6f`、`5f1bb37`)
- **Settings 页拆分**：General Settings 原本的巨型 page 按 Appearance / Updates / Privacy / ServiceMode / Hotkey / CloseBehavior / SplitTunnel 等 section 全部抽成独立 widget，primitives 统一。(`de33572`、`588a9fc`、`d38677f`、`415699e`、`941700c`、`63ee532`、`695e702`、`8d0e6fa`、`c907d9f`、`ce2a208`、`ef3112b`)
- **Core 解耦**：`CoreLifecycleManager` / `CoreHeartbeatManager` / `SubscriptionSyncService` 不再直接依赖 `proxyGroupsProvider` / `profilesProvider`，改用 ref 抽象。`subSyncIntervalProvider` 回归 `core/providers`。(`41f2fa5`、`59643bb`、`4edc18c`、`2d299d0`)
- **Nodes 抽取**：sync-and-reconnect 编排、排序过滤 helper 独立化。(`ab06b3f`、`17ec6a6`)
- **Surge modules** 按三层架构归位。(`3255441`)
- **生命周期治理**：`app_notifier._currentTopTimer` 死字段清理；analyze info 从 105 → 14（全部剩余在 main / nodes_page / test-manual 红线区）。(`11cb841`、`242ec3e`)
- **诊断工具**：新增 `ProfilesNotifier` dispose guard 回归测试；riverpod3 守护补齐。(`242ec3e`、`13e9d12`)
- **依赖维护升级（非 win32 链）**：`flutter_riverpod 2.6.1 → 3.3.1`、`fl_chart 1.2.0`、`app_links 7.0.0`、`mobile_scanner 7.2.0`、`slang 4.14.0`、`google_fonts 8.0.2`、`launch_at_startup 0.5.1`、`tray_manager 0.5.2`、`window_manager 0.5.1`、`flutter_lints 6.0.0`。(`2c9cd69`、`caa6053`、`8fe34fe`、`0959fc4`、`161b85e`、`f49855b`、`f29d452`、`803dbbf`、`51fd46e`、`00d995d`、`95cdbdb`)
- **mihomo**：v1.19.23 → v1.19.24（上游最新稳定），setup 脚本自动 apply `core/patches/`。(`a7e48ca`、`688cfee`)
- **CI 对齐**：Flutter 3.41.7 全工作流固化，release job 仅 tag 触发，发布工作流环境 floor 对齐。(`00cc871`、`c0d5c97`、`e815b44`)
- **未来准备**：`SecureStorageMigration` backup skeleton + 6 cases 测试到位，为将来的 `flutter_secure_storage 9 → 10` 升级铺路；win32 5 → 6 升级约束链写进 `docs/dependency-upgrade-plan-win32-v6.md`。**这些是内部准备，没有在启动路径被启用**。(`f167e6f`、`129d3e4`)

## 已知问题

- **Analyze 剩余 14 条 info**，全部在红线 / 不动区（`main.dart` 5、`nodes_page.dart` 5、`service_manager.dart` 2、`test/manual/` 2）。均为纯风格 lint，无功能影响。
- **`SecureStorageMigration.restoreFromShadow()` 尚未实现、`backupToShadow()` 未在启动链启用**。Android/Windows 用户在未来 fss 10 升级时如果直接切过去会丢 token，因此本次 prerelease **不做** fss 10 升级。恢复半边会和 win32 6 联动升级放在同一 PR。
- **win32 5 → 6 联动升级延后**。当前被 `file_picker 11.0.2`（尚无 win32 6 版本）和 `launch_at_startup 0.5.1`（12 个月未更新）两个上游阻塞。详见 `docs/dependency-upgrade-plan-win32-v6.md`。
- **CHANGELOG.md 停在 v1.0.14** — 历史欠账，v1.0.15 ~ v1.0.19 的 release notes 实际在 `docs/releases/` 下，`CHANGELOG.md` 本身暂不补齐（独立清理项）。

## 升级建议

- **面向用户**：这是 prerelease，建议能接受「发现 bug 能反馈」的用户安装。主要想验的是：
  - mihomo 吞吐回归后日常流量、流媒体、YouTube 是否和 v1.0.19 感知一致或更好
  - 主页模式徽章点击切换的手感与正确性
  - Windows 用户：连接状态下从托盘右键退出，进程能否在 3 秒内完全结束
  - Android / 桌面：app 切后台再回前台，连接状态不应该被自动翻回
- **已登录用户**无需重登，订阅与设置保留。
- **TUN 模式用户**：外观保持 v1.0.19 的靛蓝主题。
- **rollback**：若出现非预期行为，可回退到 v1.0.19 正式版，数据兼容。
