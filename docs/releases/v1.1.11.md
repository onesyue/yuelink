## Fixes

- **支付链路更稳** — 修复购买 / 取消 / 轮询之间的异步竞态，空支付链接现在会明确失败，不再静默卡在等待支付。
- **空支付 URL 双层防御** — 后端返回非免费订单但缺少支付链接时直接进入失败态；轮询耗尽后若原始支付链接为空，也会明确失败。
- **取消订单刷新一致** — 当前订单取消后同步刷新订单历史，避免订单列表继续显示旧状态。

## Improvements

- **连接修复页拆分** — 诊断日志、网络探测、修复动作和 TUN 状态 widget 已拆出独立模块，保留原视觉，同时补上 service/actions/widget 回归测试。
- **ServiceManager 探活补测试** — 桌面 helper 安装态、就绪态、IPC 可达性和平台分支增加测试切口；安装 / 卸载脚本本轮未重写。
- **Riverpod 3 迁移完成** — 全仓 `flutter_riverpod/legacy.dart` 清零，旧 `.notifier.state =` 写法清零，状态 provider 统一迁到 `NotifierProvider`。
- **ConfigTemplate 拆分** — 配置模板处理拆成 DNS / TUN / rules / scalar / provider-proxy 等 transformer，并新增 golden lock，保证重构前后输出稳定。
- **启动逻辑拆分** — `main.dart` 的设置加载、运行时 provider bootstrap、单实例守卫拆到 `lib/app/bootstrap/`，降低入口文件复杂度。
- **技术债清单落盘** — 新增 `docs/debt.md`，把已知风险债、形态债、平台覆盖限制和不应误判的非债项集中记录。
