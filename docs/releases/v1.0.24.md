## What's New

- **签到日历 native 月视图** — 设置 → 订阅相关 → 签到日历，或仪表盘新加的「📅 签到日历」独立卡片直接进入。emoji 网格表达每天签到状态（✅ 已签 / ⭐ 补签 / ⛔ 断签 / 🔴 今天未签 / ⏳ 未来），头部摘要含连签天数 / 本月已签 / 距下个里程碑（7/14/30/60/90 天），支持月份左右切换 + 下拉刷新。
- **补签卡 native** — 断签当天日历底部出现「⭐ 用 25 积分补昨天」按钮,确认弹窗实时显示当前积分余额 + 后端权威价格,扣积分 / 写历史 / 维持 streak 一气呵成。
- **签到卡副标题显示连签天数** — 签到完仪表盘「已签到 · 连签 N 天」(≥2 天才显示,首签不啰嗦);仪表盘签到卡多一个日历图标按钮一键直跳月历。
- **iOS 安装方式说明页** — 设置 → 关于 → iOS 安装方式,列三种侧载方式(AltStore / TrollStore / IPA 直装)的 pros / cons / 操作步骤,明确告知巨魔不支持 VPN。
- **签到日历 / 补签 / iOS 安装方式 全 i18n 中英** — 56 个新 key,英文为产品级文案,周一到周日缩写按 iOS HIG 用 3 字母(Mon / Tue / ... / Sun)。

## Bug Fixes

- **iOS 巨魔(TrollStore)「提示连接成功但实际无网络」** — 根因是 NetworkExtension 仍依赖 Apple 签发的 provisioning profile,巨魔 IPA 拿不到这条信任链,系统让 PacketTunnel 看起来启动但不放行任何包。这版加 fail-loud 检测:连接成功后 10 秒内被系统中止,自动判定 entitlement 可疑,弹出全屏「iOS 安装方式」引导页 + AltStore / SideStore 推荐,不再静默失败。
- **`CheckinResult.streak` 字段补齐** — 之前签到结果模型里漏解析 streak,导致依赖此字段的 UI 拿不到值。补全后副标题能稳定渲染连签天数。

## Server-side

YueLink Checkin API 同步加两个 endpoint,签到日历和补签卡依赖。已部署,无需用户操作:

- `GET /api/client/checkin/history?month=YYYY-MM` — 月历数据,把 `gambling_points` 和 `sign_card_cost` 一并发回省一次往返。
- `POST /api/client/checkin/resign` — 25 积分补昨天 + 维持 streak。
- 新建 `app_sign_history` 表(独立于 Telegram bot 通道),保留双签政策。
