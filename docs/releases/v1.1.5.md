## What's Improved

- **设置页真实页面布局修复** — 新增真实设置页截图矩阵，覆盖外观、连接、启动、路由、主题、关闭窗口和服务模式等设置项，修复窄屏和大字号下 segmented / dropdown 控件与左侧文字挤压、换行或重叠的问题。
- **中英文与大字号适配** — “跟随系统”“最小化到托盘”“路由模式”等设置项在 320 / 360 / 768 / 1440 宽度、light / dark、text scale 1.0 / 1.3 / 1.6 下通过布局检查。
- **桌面 TUN 生命周期收敛** — TUN start / stop / restart / hot-switch / recovery stop 入口进一步收敛到生命周期管理流程，减少快速切换、恢复流程和模式切换时出现状态不同步的风险。
- **连接修复诊断增强** — 诊断包 `structured_snapshot` 增加 `platform` 和 `cleanup_ok` 字段，并继续输出 controller、interface、route、DNS、running verification 等结构化状态，便于定位权限、driver、路由和 DNS 问题。
- **诊断导出可靠性** — 保留此前中文 Windows 日志导出修复：`event.log` 非 UTF-8 字节不会导致导出失败，`netsh` / `ipconfig` 中文输出不再因 codepage 导致乱码。

- 直接覆盖安装即可，所有平台。
- 如果打开代理后仍显示“连接失败”，请进入 设置 → 连接修复，查看具体归因并导出诊断包。

## Known Limits

- 本版本是 **UI 布局、TUN 生命周期一致性与诊断增强版本**。
- 节点池稳定性仍在治理中，本版本不承诺默认组节点质量已经完成治理。
- Claude / ChatGPT 出口风控仍可能存在，本版本不承诺 AI 出口池已经完成治理。
- 桌面 TUN 已增强状态一致性与诊断能力，但 Windows / macOS / Linux 全矩阵和 24h soak 仍需继续验证；不要把本版本描述为 “TUN 已彻底稳定”。
