## New

- **iOS 局域网设备直连** — AirPlay 投屏、HomeKit 智能家居、AirDrop 互传、群晖 / QNAP NAS 局域网访问、路由器管理界面 192.168.x.1，开着 yuelink TUN 时不再绕一圈 mihomo，**直接走本机网络**。Continuity / Universal Clipboard / Sidecar 同样恢复。和 Android 之前已经做的 `PUBLIC_IPV4_ROUTES` 拆路由对齐，iOS `NEPacketTunnelProvider` 加 7 段 IPv4 `excludedRoutes`（10/8、127/8、169.254/16、172.16/12、192.168/16、224/4、255.255.255.255）。

- **Windows 局域网兼容模式开关** — 桌面"高级 → 局域网兼容模式"。开启后 Windows TUN 的 `strict-route` 切到 `false`，可访问内网 SMB 共享 / 网络打印机 / 远程桌面到内网机 / NAS 管理页。默认关（保留更严的防泄漏 strict-route: true）。

- **Android Private DNS 冲突提示** — 当系统级 Private DNS 设到 `hostname` 模式（用户在系统设置手动指定了 DoT 服务器，例如 `1dot1dot1dot1.cloudflare-dns.com`），Dashboard 顶部出现温和提示。这是 Android 唯一会真正绕过 yuelink TUN 的 DNS 接管的模式。`opportunistic`（三星等默认）和 `off` 不弹提示。

- **Dashboard 连通性体检** — 一键测 Apple / GitHub / Google / YouTube / Cloudflare / Anthropic 6 个站点的可达性 + 延迟。装好 yuelink 后第一时间能看到"是否真能访问 GitHub / 海外 AI"，不用自己开浏览器试。

- **导出诊断报告** — 设置 → 高级（连接模式 = TUN）→ 导出诊断报告，生成 markdown 含模式 / 端口 / 启动诊断 / 系统代理 / Service Mode / Private DNS / TUN bypass 11 板块。**不**包含订阅 URL / token / 节点地址 / 当前出口 IP / 用户名路径 —— 整段直接贴 issue 安全。

- **Windows 诊断 PowerShell 脚本** — 设置里点"复制 Windows 诊断脚本"，粘贴到 Windows PowerShell 执行，输出 9 项检查 markdown：Wintun 驱动 / Meta-YueLink 网卡 / 默认路由 / 系统 DNS 是否被 TUN 接管 / 防火墙 / HKCU 系统代理 / Service Mode helper / mihomo external-controller 可达性 / underlying transport metric。只读，不改任何系统状态。

- **桌面"托盘空闲标志"开关（实验性）** — 窗口隐藏托盘 N 分钟后将一个 Riverpod flag 设为 true，供未来按页 opt-in 释放重型缓存。当前**无消费者**实际释放资源 —— 仅作为接线点，托盘 + mihomo 始终正常运行。文案已说明这是实验性。

## Fixes

- **Emby 视频"继续播放"从断点起播（之前点继续仍从 0 起）** — `_player.open()` 默认 `play: true` 会立刻从 0 自动播放，而"上次播放到 12:34 — 继续 / 从头"对话框要先拉一次 Emby `/Users/.../Items/...?Fields=UserData` API（经代理 ~200-500 ms）才弹。等用户点"继续"时视频已经从 0 跑了 1-3 秒，后面 `_player.seek(posSeconds)` 经常被自动播放路径覆盖，落不下。改成 `open(play: false)` → await 对话框决策 → seek 在 paused 状态下 100% 落地 → `play()` 从断点起播。

- **AI / cf-fronted 服务防 DNS 泄漏** — 之前 yuelink 只靠 mihomo `geosite:geolocation-!cn` catch-all 把 AI 域名定向到海外 DoH。但 geosite 数据库通常滞后 3-7 天，新出的 AI 服务（`cursor.com` / `bolt.new` / `v0.dev` / `replit.com` / `runwayml.com` 等）在数据库更新前会落到 CN DoH（AliDNS / 腾讯）—— 解析到 CN CDN → AI 服务无法访问 + 用户的 AI 使用画像泄漏给 CN DNS 商。本版本在 `nameserver-policy` 显式枚举 60+ AI 域名（OpenAI / Anthropic / Google AI / xAI / Meta / Perplexity / HuggingFace / Mistral / Cursor / Cohere / Replit / Suno / ElevenLabs 等），不再依赖 geosite 数据库版本。

- **国内银行 / 运营商认证 / 支付 App 在 yuelink 连接时登录失败** — 之前 fake-IP 模式下 `cmpassport.com`（中国移动一证通）/ `wosms.cn`（中国联通短信网关）/ `cmbchina.com`（招商银行）/ `pingan.com.cn`（平安）/ `alipay.com` 等域名会被误判为外网域名拿 fake-IP，导致 App 内嵌 WebView SSL 校验失败。`fake-ip-filter` 加 33 项中国本土关键服务（运营商认证 / 银行 / 支付 / 出行 / 企业内网 / Active Directory），这些域名走真实 DNS。

- **fake-ip-filter 注入跨子段误判 silent bug** — 之前 `dns_transformer` 内部判断"某域名是否已在 fake-ip-filter"用整个 `dns:` 段做字符串 contains，会被同段 `nameserver-policy` 已有的 `geosite:cn:` / `+.openai.com:` 撞中而**跳过插入**。现象：升级后看 dump 配置，`fake-ip-filter` 表面"看起来加了 geosite:cn"实际没生效（只在 nameserver-policy 那行匹配上了）。子段去重正则容忍注释和空行作为列表延续，杜绝静默跳过。

- **Windows TUN 热切换 vs 启动配置漂移** — `core_lifecycle_manager` 里桌面热切换走全重启走 `TunTransformer.ensureDesktopTun`（已是完整配置）；mobile 路径之前 `patchConfig({tun: { auto-route: true, auto-detect-interface: true, ... }})` 的字段在 Android 14+ netlink 禁用后是错误的（与 startup 时 `injectTunFd` 的 `auto-route: false` 冲突）。简化为 `{enable: bool}`，其它字段保留 startup 配置不动。

- **直连流量也遵循 nameserver-policy（mihomo 1.18+）** — 加 `direct-nameserver-follow-policy: true`。之前用户自定义企业内网 policy `+.corp.example.com -> 192.168.1.53` 对 DIRECT 流量不生效，`+.corp.example.com` 会泄漏给 alidns/doh.pub。

## Internal

- **DNS 配置单一真源（`dns_policy_catalog.dart`）** — 60+ AI 域名 / 33 国内银行运营商 / 13 OEM 连通性 / 5 ECH 探测域名 / 海外 DoH endpoint 等清单全在这一份 const 里，`dns_transformer` 和 `rules_transformer` 共享读取，杜绝两边漂移。catalog 自身有 11 项不变量测试（无重复 / 命名约定 / 子集关系 / OEM 覆盖率）。

- **`TunTransformer.buildDesktopTunYaml` 纯函数化** — `windowsLanCompatibilityMode` 经 `ConfigTemplate.process` 透传到 builder，transformer 无 `SettingsService` / Riverpod 依赖。一致性测试断言 `ensureDesktopTun output contains buildDesktopTunYaml output verbatim`，hot-switch / cold-start 不会 drift。Windows strict-route 分支用 `TunPlatform` 测试可注入 —— CI 在非 Windows 上也能覆盖 Windows 默认 `strict-route: true` 回归。

- **诊断报告 PII redaction** — `homeDir=` / macOS `/Users/<n>/` / Linux `/home/<n>/` / Windows `C:\Users\<n>\` / Android `/data/user/N/<pkg>/` / 32+ hex `secret=` 六类全部 strip。macOS Application Support 路径含空格的边界 case 单独覆盖（`homeDir=.+?(?=,\s|\n|$)` lookahead 模式）。10 个 redaction 测试。

- **mihomo 月度跟版 governance** — `governance/mihomo-bump-cadence.md` 每月第一周流程 + Alpha 锁定 invariant + 与 sniffer benchmark 的 SHA 关联 + cadence note 模板。

- **CLAUDE.md MTU 注释 iOS-only 限定** — 之前 stale 注释 "keep mtu: 1500" 没限定平台，但事实是 Android + Desktop 都 9000、iOS 才 1500。
