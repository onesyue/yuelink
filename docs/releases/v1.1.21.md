## New

- iOS 局域网设备直连：AirPlay 投屏、HomeKit 智能家居、AirDrop 互传、群晖 / QNAP NAS 局域网访问、路由器管理界面 192.168.x.1，开着 yuelink TUN 时不再绕一圈 mihomo，直接走本机网络。Continuity / Universal Clipboard / Sidecar 同样恢复。

## Fixes

- 悦视频"继续播放"从断点起播：之前会从 0 起播再被对话框打断，点继续后断点 seek 经常被自动播放路径覆盖落不下。改为先暂停打开 → 等用户决策 → 在暂停状态下 seek → 再播放，断点稳定。
- AI / cf-fronted 服务防 DNS 泄漏：之前只靠 mihomo geosite 兜底把 AI 域名路由到海外 DoH，但 geosite 数据库滞后 3–7 天，新出的 AI 服务（cursor.com、bolt.new、v0.dev、replit.com、runwayml.com 等）在数据库更新前会落到国内 DoH。本版在 nameserver-policy 显式枚举 60+ AI 域名（OpenAI / Anthropic / Google AI / xAI / Meta / Perplexity / HuggingFace / Mistral / Cursor / Cohere / Replit / Suno / ElevenLabs 等），不再依赖数据库版本。
- 国内银行 / 运营商认证 / 支付 App 在 yuelink 连接时登录失败：之前 fake-IP 模式下 cmpassport.com / wosms.cn / cmbchina.com / pingan.com.cn / alipay.com 等域名会被误判为外网域名拿 fake-IP，导致 App 内嵌 WebView SSL 校验失败。fake-ip-filter 加 33 项中国本土关键服务（运营商认证 / 银行 / 支付 / 出行 / 企业内网 / Active Directory）走真实 DNS。
- 直连流量也遵循自定义 DNS 策略（mihomo 1.18+）。之前用户在覆盖配置里加的企业内网域名 → 内网 DNS 服务器策略对 DIRECT 流量不生效，会泄漏给国内 DoH。

## Upgrade Notes

- 拉新订阅一次即生效。已登录用户无需重登，订阅、设置、节点选择全部保留。
- iOS 用户从 v1.1.20 升上来后先确认 AirPlay 投屏 / 群晖局域网访问能用，再继续日常使用。
- 没有改 mihomo 内核版本、TUN MTU（Android + Desktop 9000、iOS 1500）、节点选择口径。
