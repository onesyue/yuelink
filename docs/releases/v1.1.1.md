## What's New

- **网络切换自动恢复** — Android、iOS、macOS、Windows、Linux 都补齐了网络变化感知。Wi-Fi 切蜂窝、蜂窝切 Wi-Fi、切换到另一个 Wi-Fi、桌面唤醒后网络变化，都会自动触发代理恢复，不再依赖手动关开代理。
- **桌面 TUN 首次安装入口** — 设置页和连接修复页都能直接安装 / 更新桌面 Service Mode，首次使用不会因为状态刷新失败而看不到安装按钮。
- **桌面网络守护** — macOS / Windows / Linux 会轮询真实物理网络接口，过滤 TUN、Docker、虚拟网卡等噪声，发现网络变化后自动校验系统代理和 TUN DNS。

## Bug Fixes

- **Android VPN handoff** — 重新排序 `underlyingNetworks`，默认网络变化后刷新 DNS，并覆盖同类型网络变化，减少 Wi-Fi / 蜂窝互切后的断流。
- **Apple network path** — iOS / macOS 使用 `NWPathMonitor` 通知 Flutter 层恢复代理。
- **macOS TUN DNS** — 网络变化后会重应用 TUN DNS，同时保留原始 DNS 备份，停止代理后能正确还原。
- **mihomo 控制通道** — 本地 REST API 与 WebSocket 强制 DIRECT，避免系统代理把 `127.0.0.1` 控制请求绕回 mixed-port。
- **恢复策略** — 网络切换后分阶段重试，仍失败时执行轻量重启，减少“必须手动重开”的情况。

## Upgrade Notes

- 已登录用户无需重登，订阅与设置自动保留。
- Android：直接覆盖安装新版 APK；建议重点验证 Wi-Fi / 蜂窝互切。
- macOS：DMG 直接覆盖；TUN 模式首次使用可在设置或连接修复页安装 Service Mode。
- Windows / Linux：直接覆盖；切换网络或系统唤醒后会自动检查并修复代理状态。
- iOS：TrollStore / AltStore / SideStore 重装新 IPA 即可。
