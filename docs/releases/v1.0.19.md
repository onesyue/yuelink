正式版，发布日期 2026-04-19。

### 发布亮点

- **QUIC 策略改为可配置**：原来硬编码的全局 `UDP:443 REJECT` 改成 `off / googlevideo / all` 三档策略，默认 `googlevideo`——只对 YouTube 视频 CDN 做 TCP 回落。选 `off` 可以让 MyTV Super 等依赖 HTTP/3 的港区流媒体正常工作；`all` 保留旧行为供排障。入口在「设置 → 连接 → QUIC 策略」，中英文双语。(`89192e4`)
- **experimental 回归 mihomo 上游默认**：不再默认注入 `quic-go-disable-gso` / `quic-go-disable-ecn`。这两项在合规网络上每项约减 hy2/QUIC 吞吐 20–30%，上游默认就是 false，本次对齐。订阅自带的 `experimental` 段保留不动。(`89192e4`)
- **桌面 TUN MTU 统一到 1500**：新增共享常量 `AppConstants.defaultTunMtu = 1500`，桌面冷启动 TUN、Android/iOS fd TUN、热切换 PATCH 都走同一个值。修复旧版「冷启动 1500 / 热切换 9000」的不一致——同一个用户在两种进入方式下吞吐不同的问题。(`89192e4`)
- **TUN 模式运行态视觉高亮**：运行中的 TUN 模式，Dashboard 的连接卡从翠绿切到靛蓝（`#6366F1`），边框 / 状态点 / 连接文字 / 电源按钮 / 路由徽章 / 模式徽章整组同步变色。系统代理模式外观不变，一眼可区分当前是哪种模式。(`1c42f8b`)
- **Tray 菜单顶部状态项可直接点击**：右键 tray 菜单顶部的「YueLink · 已连接 · 节点名」以前是不可点击的 label，现在点一下直接拉起主窗口。原「显示窗口」条目保留；macOS 左键 tray 仍弹菜单（原生 menubar 习惯），Windows 左键 tray 仍直接打开窗口。(`1c42f8b`)
- **其它修复**：收紧 TUN MTU 测试的平台判定，和产品代码路径一致（`9324c28`）；修正 Build workflow 对 `pre` / `*-pre` tag 的 prerelease 判定，以后带 `-pre` 后缀的 tag 会自动走预发布通道，不再需要手工 edit 兜底（`11fbe2b`）。

### 校验说明

- 每个正式产物都会附带一个同名 `.sha256` 文件，用于校验下载完整性。
- Android 用户优先下载 `android-universal.apk`；只有明确知道设备架构时，才建议改下分 ABI 包。
- Windows 默认推荐 `setup.exe`；只有不方便安装或需要绿色版时，再使用 `portable.zip`。
- iOS 包为侧载用途，不走 App Store 分发。

### 版本一致性

本次正式版完成版本号、tag、产物命名三者对齐：

- `pubspec.yaml`：`1.0.19+119`
- Git tag：`v1.0.19`（annotated tag，指向 release bump commit `29692c6`）
- 产物命名：`YueLink-1.0.19-*`

此前发布的 `v1.0.19-pre` 预发布 tag 指向的是 bump 之前的 commit（`1c42f8b`），所以那一批产物文件名仍保留 `YueLink-1.0.18-*`——这是 pre-release 阶段的预期状态，不是缺陷。

### 升级提示

- **全量自动生效**：QUIC 默认策略是 `googlevideo`，行为上等价于 v1.0.18 的 YouTube 窄规则，不会伤及 MyTV Super 等 QUIC 依赖型流媒体；升级后首次启动即可享受。
- **已登录用户**无需重登，订阅与设置自动保留；之前在 v1.0.18 调过的设置项不受影响。
- **TUN 模式用户**：外观会由翠绿切到靛蓝，这是预期变化，不是 bug。
